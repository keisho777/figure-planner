## サービス概要
本サービスは、フィギュアをよく購入・予約する人、フィギュアを趣味として楽しんでいる人向けの管理アプリです。
受注生産による「将来の支払い」や「発売時期」、「購入・予約したフィギュアの個数」を可視化し、
計画的にフィギュアを購入・管理できるようにします。

## このサービスへの思い・作りたい理由
私はフィギュアが好きで、日頃から購入することが多くあります。
しかしフィギュアは、ほとんどが受注生産という特殊な販売形態であり、実際に商品が届くのは購入から1年後になることも珍しくありません。
私自身、半年ほど経つと「何を予約したのか」を忘れてしまうことがあります。
この状況で特に不安に感じた点が、2点あります。
1点目は、支払いタイミングの問題です。
店舗にもよりますが、クレジットカード決済の場合、購入時ではなく商品が店舗に入荷したタイミングで引き落としが行われるケースが多くあります。
つまり、購入から1年後に引き落としが発生することになります。
また、フィギュアは高価な商品が多いため、予期せぬ高額請求に焦ってしまったり、人によっては生活に支障が出てしまう可能性もあると感じました。
なお、フィギュアは基本的にキャンセル不可の商品が多いです。
2点目は、知らないうちにフィギュアが増え、飾る場所や保管スペースが不足してしまう問題です。
フィギュアは受注生産のため在庫販売がなく、予約を逃すと再販されない限り入手が非常に困難になります。
そのため、つい購入してしまいがちですが、何も考えずに予約を重ねると、
すでに所持しているフィギュアで飾る場所がなくなり、結果として箱に入れたままという悲しいことになってしまうことがあります。
また、フィギュアの箱は捨てずに保管するのが一般的で、市場においても箱は商品の一部として扱われます。
そのため、箱を安易に処分することができず、購入数が増えるほど生活スペースを圧迫してしまう恐れもあります。
これらの問題を事前に把握し、対策できるようにしたいと考えたことが、本サービスの背景です。

## ユーザー層について
対象ユーザーは、フィギュアをよく購入・予約する人や、フィギュアを趣味として楽しんでいる人です。
フィギュアは受注生産が多く、購入から商品到着までに長い期間（約１年ほど）が空くという特徴があります。
そのため、複数のフィギュアを予約していると、「何を予約したのか」「いつ支払いが発生するのか」を把握しづらくなりがちです。
また、フィギュアは高価な商品が多く、購入数が増えるほど将来的な支払い負担や保管・展示スペースの問題が発生します。
これらの課題は、フィギュアを継続的に購入する人ほど強く影響を受けます。
そのため、本サービスの対象ユーザーとしています。

## サービスの利用イメージ
ユーザーは、フィギュアを予約・購入したタイミングで本サービスに情報を登録します。
登録したフィギュアは一覧で管理でき、発売月や支払い予定を事前に把握することができます。
これにより、「いつ・どのフィギュアが届くのか」「今後いくらの支払いが発生するのか」を
事前に可視化でき、突然の高額なクレジットカードの引き落としや、
飾るスペースを超えたフィギュアの購入を防ぐことができます。

## ユーザーの獲得について
想定ユーザーは、フィギュアを複数予約・購入しているフィギュアファンです。
このようなユーザーは、ブログなどでは感想を述べず、X（旧Twitter）で「届いた！めっちゃ綺麗！」といった
一言コメントのように発信するケースが多いため、SNSでの情報発信や、開発者自身の利用体験をもとにした発信を通じて
サービスを知ってもらうことを想定しています。

## サービスの差別化ポイント・推しポイント
Q.購入記録アプリなら家計簿アプリでいいのでは？

A.家計簿アプリは支払い終えたものを記録することに向いていますが、まだ支払っていないものに対しては向いていないかと思います。
家計簿アプリに未来の支払いを記録すると、例えば１月にフィギュアを購入したとします。
これを１月に記録してしまうと、既に支払ったと勘違いしてしまいます。
それに対して、本サービスは購入日と発売月の２つを用意し、発売月が近づいてきたらメールで通知するというようにします。
もちろん購入金額も記録可能です。

Q.未来に対しての通知ならカレンダーやリマインダーでもいいのでは？

A.たしかに、未来の支払いを知るという点では問題ないかと思います。
ただ、「〇月っていくら払う可能性があるんだ？」や「〇月っていくつの受注生産品が届くんだ？」ということを確認したいとき、
即座に分からないという欠点があります。それに対し、本サービスは指定の月の支払額の合計、
購入・予約済みのフィギュアの個数などを表示し、カレンダーやリマインダーではできないことを実装し差別化します。

Q.家計簿アプリで未来の月に支払い記録を付けたらいいのでは？

A.多くの家計簿アプリは「未来の支払い」を前提に作られていないはずなので、そのように利用すると不自然に感じる可能性があります。
また、まだ支払っていないのに赤字みたいな状況になったり、何を購入したのかという細かいところがわかりません。
本サービスは未来の支払いを管理するために作られるものなので、前述のような違和感はありませんし、何を購入したのかの
検索も可能です。

## 機能候補
- MVP機能
  - ログイン機能（メールアドレス＆パスワード、LINEログイン）
  - 予約フィギュアのCRUD機能
  - 購入したフィギュアの一覧表示、並び替え機能
  - 未払い/支払い済みのステータス変更機能
  - 検索時のオートコンプリート機能
  - フィギュアの発売月が近づいたときのメール通知機能
  - 月別の支払い予定額表示機能

- MVP後
  - 月別の支払い予定額を棒グラフでも表示
  - フィギュアの発売月が近づいたときのLINE通知
  - レスポンシブ対応

## 使用する技術スタック
- フレームワーク：Ruby on Rails 7
- データベース：PostgreSQL
- デプロイ先：Render
- 使用予定のライブラリ
  - 認証関連：Devise、OmniAuth、omniauth-line-login
  - フロントエンド：Hotwire（Turbo/Stimulus）、autocomplete-stimulus、Chartkick
- 外部サービス：LINEログイン、Messaging API
